# 领域上下文

## 目标

构建一个面向运营的数字图书服务：管理员维护图书库，普通用户通过兑换码将图书加入个人书架并持续阅读。

## 术语表

- 图书（Book）：平台可被兑换和阅读的 PDF 资源及其元数据，不等同于“用户上传文件”。
- 书架（Bookshelf）：用户已持有图书的集合，包含阅读状态。
- 兑换码（Redeem Code）：由管理员为某本图书生成的一次性凭证，用于授予用户持有权。
- 持有权（Ownership）：用户兑换后获得的长期访问资格；当前版本默认不可主动删除。
- 图书状态（Book Status）：
  - `active`：可生成新兑换码、可被新用户兑换
  - `inactive`：暂停运营使用，不建议新兑换
  - `archived`：归档状态，已持有用户仍可阅读
- 管理员（Admin）：具备图书管理与兑换码管理权限的内部角色，需数据库授权。
- 普通用户（User）：可注册登录并管理个人阅读行为的外部角色。

## 业务边界

- 用户侧入口是“兑换入书架”，不是“私有 PDF 上传器”。
- 管理员上传行为服务于公共图书库运营。
- 阅读进度、书签、最后阅读时间属于书架阅读状态，而非图书元数据。

## 一致性约束

- 对外文档应统一使用“图书 / 兑换码 / 书架”术语。
- 若文档出现“PDF 上传（用户自有）”语义，应视为历史模型并标记归档。
